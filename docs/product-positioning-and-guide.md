# Prompt Vault 产品边界逻辑、定位与使用指南

## 一、产品边界逻辑与定位

Prompt Vault 是一个本地优先的 Prompt 资产管理工具。它的核心不是“帮用户生成 Prompt”，也不是“替代大模型平台”，而是帮助 AI 高频团队把已经产生、正在迭代、值得复用的 Prompt 沉淀为可搜索、可版本化、可分享、可控隐私的团队资产。

### 1. 用户定位 —— “为谁”

#### 1.1 目标用户是谁？

核心目标用户是 AI 高频团队中的 Prompt 使用者和管理者，包括：

- 产品经理：沉淀用户研究、竞品分析、需求拆解、PRD 辅助类 Prompt。
- 运营和市场团队：沉淀文案生成、活动策划、内容改写、渠道投放类 Prompt。
- 研发和数据团队：沉淀代码解释、SQL 生成、日志分析、测试用例生成类 Prompt。
- 设计和内容团队：沉淀视觉描述、创意发散、风格约束、内容结构化类 Prompt。
- 团队负责人或知识库维护者：整理团队最佳实践，避免优质 Prompt 散落在聊天记录和个人笔记里。

非核心目标用户：

- 只偶尔使用 AI、没有 Prompt 复用需求的轻量个人用户。
- 主要诉求是在线调用多模型、自动执行工作流的用户。
- 需要企业级 SSO、权限审批、审计日志、实时多人协作的大型组织。
- 只想购买 Prompt 模板市场内容，而不是管理自有 Prompt 资产的用户。

#### 1.2 用户关键特征是什么？

目标用户通常具备以下特征：

- 高频使用 AI：每天都会在 ChatGPT、Claude、本地模型或企业模型中反复使用 Prompt。
- Prompt 有复用价值：同一类任务会反复发生，例如周报生成、竞品分析、需求拆解、营销文案。
- Prompt 会持续迭代：用户经常微调表达、变量、限制条件、输出格式和模型建议。
- 团队存在共享需求：优秀 Prompt 不应只存在于个人聊天记录、Notion 页面或临时文档里。
- 对隐私敏感：部分 Prompt 包含业务策略、客户信息、内部方案、产品路线或行业 Know-how。
- 管理习惯偏“资产化”：希望 Prompt 像文档、代码、设计组件一样被沉淀、检索、复用和追踪历史。

### 2. 场景定位 —— “在哪用”

#### 2.1 用户在什么情况下想到这个产品？

用户会在以下场景中想到 Prompt Vault：

- 找不到之前效果很好的 Prompt，只能翻聊天记录、Notion、飞书文档或本地记事本。
- 修改 Prompt 后发现旧版本更好，但没有历史记录可以恢复。
- 团队成员各自维护 Prompt，格式不统一，复用成本高。
- 新成员加入团队后，不知道哪些 Prompt 是经过验证的高质量版本。
- Prompt 中包含敏感业务内容，不适合放到第三方云端工具。
- 想把某个 Prompt 标准化为模板，例如通过 `{{产品名称}}`、`{{目标用户}}` 变量进行复用。
- 需要把一组 Prompt 分享给团队成员，但又不想引入复杂的云同步或账号体系。

#### 2.2 是高频日常工具，还是特定场景解决方案？

Prompt Vault 的定位是“AI 高频团队的日常资产管理工具”，不是一次性的 Prompt 生成器。

它适合被放在团队日常工作流中：

- 写 Prompt 前，先搜索是否已有可复用模板。
- 使用 Prompt 时，复制原文或填充变量后复制。
- 优化 Prompt 后，保存为新版本并写明修改说明。
- 团队沉淀最佳实践时，标记收藏或最佳版本。
- 跨成员共享时，导出标准分享包并由对方导入。

边界上，它不承担以下场景：

- 不直接调用大模型生成结果。
- 不做自动化 Agent 流程编排。
- 不做团队实时协同编辑。
- 不做公有云 Prompt 市场。
- 不做企业知识库全文替代。

### 3. 价值定位 —— “解决什么问题”

#### 3.1 产品提供的核心价值是什么？

Prompt Vault 提供的核心价值是：把 Prompt 从零散文本变成可管理的本地团队资产。

具体价值包括：

- 集中管理：把散落在聊天记录、Notion、文档、记事本里的 Prompt 统一收纳。
- 快速检索：通过关键词、标签、用途、模型建议、收藏状态快速找到 Prompt。
- 版本可追溯：每次编辑都生成版本记录，可查看差异、标记最佳版本、恢复旧版本。
- 模板化复用：支持 `{{变量名}}` 形式，让 Prompt 从“一段文本”变成“可填充模板”。
- 本地隐私：数据默认保存在本地 SQLite，不依赖第三方云端。
- 标准化分享：通过导入/导出包共享 Prompt、标签和版本历史，降低团队复用门槛。

#### 3.2 用户愿意投入什么成本来获得价值？

用户愿意投入的成本主要不是金钱，而是较低的整理成本和使用习惯迁移成本：

- 初始整理成本：把已有高质量 Prompt 导入或手动录入。
- 标准化成本：为 Prompt 补充标题、描述、标签、用途和模型建议。
- 维护成本：每次优化 Prompt 时写一句修改说明。
- 协作成本：团队约定标签命名、最佳版本标记和分享包导入规则。

用户愿意投入这些成本的前提是：

- 找 Prompt 的时间明显减少。
- 好 Prompt 不再因为修改而丢失。
- 新成员能更快复用团队经验。
- 敏感 Prompt 不需要默认上传到第三方平台。

### 4. 竞争定位 —— “为什么是我们”

#### 4.1 和替代方案相比，差异化在哪？

常见替代方案及差异：

- Notion / 飞书文档：适合写文档，但版本对比、变量填充、结构化标签和 Prompt 复用体验弱。
- Excel / 表格：适合列表管理，但长文本编辑、版本历史和复制使用体验不佳。
- 本地记事本 / Markdown：隐私好，但检索、标签、版本恢复和团队标准化弱。
- 聊天记录收藏：保存方便，但难以检索、难以分类、难以沉淀为团队资产。
- 云端 Prompt 管理平台：协作更强，但敏感业务 Prompt 存储在第三方云端有隐私风险。
- 多模型调用平台：执行能力强，但重点在模型调用，不是本地资产管理和版本治理。

Prompt Vault 的差异化定位：

- 本地优先：默认数据在用户设备上，降低敏感 Prompt 外泄风险。
- 资产管理优先：围绕收藏、标签、搜索、版本、恢复和导入导出设计。
- 团队轻协作：先用标准分享包完成复用，不强制引入账号、云同步和权限系统。
- Prompt 专用结构：比普通文档更懂 Prompt 的变量、模型建议、用途和版本差异。

#### 4.2 这个差异化是否可持续？

差异化可持续，原因是：

- Prompt 会越来越多，资产管理需求会持续增长。
- AI 团队对隐私和可控性的要求会越来越高。
- Prompt 迭代和复用会逐渐从个人技巧变成团队知识管理问题。
- 本地优先和轻协作的组合能覆盖一批不愿上云、但又需要团队复用的用户。

后续可持续增强方向：

- 本地加密库：进一步强化敏感 Prompt 管理。
- 私有同步服务：面向团队内网或自托管部署。
- 语义搜索：让用户通过意图找到 Prompt，而不只依赖关键词。
- Prompt 质量评分：记录使用效果、适用模型、输出稳定性。
- 权限和审计：服务企业级团队治理。

## 二、产品边界总结

### 1. 我们做什么

- 管理 Prompt 资产。
- 搜索和筛选 Prompt。
- 记录 Prompt 版本。
- 对比和恢复历史版本。
- 支持模板变量填充。
- 支持本地存储。
- 支持标准导入导出。
- 支持团队轻量共享。

### 2. 我们暂时不做什么

- 不直接调用大模型。
- 不做 Agent 自动执行。
- 不做云端 SaaS 账号体系。
- 不做实时多人编辑。
- 不做企业 SSO 和权限审批。
- 不做 Prompt 交易市场。
- 不做完整知识库替代品。

### 3. 一句话定位

Prompt Vault 是一个面向 AI 高频团队的本地优先 Prompt 资产库，用搜索、版本、变量模板和导入导出，帮助团队把 Prompt 从零散文本沉淀为可复用、可追溯、可控隐私的工作资产。

## 三、产品使用指南

### 1. 启动产品

开发模式：

```bash
npm run dev
```

一键打开本地网站：

```bash
npm run open:web
```

Windows 下也可以双击项目根目录的 `Prompt Vault Web.cmd`，脚本会启动本地服务并自动打开浏览器。

构建后运行：

```bash
npm run build
npm start
```

默认访问地址：

```text
http://127.0.0.1:4317
```

本地数据默认保存在：

```text
Windows: %APPDATA%\Prompt Vault\prompt-vault.sqlite
macOS: ~/Library/Application Support/Prompt Vault/prompt-vault.sqlite
Linux: ~/.local/share/prompt-vault/prompt-vault.sqlite
```

Web 版和 Electron 桌面端默认使用同一份本地 SQLite 数据库；首次启动时，如果共享数据目录还没有数据库，会从项目内旧的 `data/prompt-vault.sqlite` 迁移一次。也可以通过 `PROMPT_VAULT_DB` 环境变量指定自定义数据库路径。

### 2. 首页结构

产品主界面分为三块：

- 左侧筛选区：搜索框、收藏筛选、标签筛选、隐私提示。
- 中间资产列表：展示 Prompt 标题、描述、用途、模型建议、标签和收藏状态。
- 右侧详情区：展示 Prompt 正文、变量填充、版本历史和版本对比。

### 3. 创建 Prompt

操作步骤：

1. 点击右上角“新建 Prompt”。
2. 填写标题，例如“新品发布文案”。
3. 填写描述，用一句话说明这个 Prompt 的用途。
4. 填写用途，例如“营销运营”“市场调研”“内部评审”。
5. 填写模型建议，例如“GPT-5.4”“本地模型”“Claude”。
6. 在正文中输入 Prompt 内容。
7. 如需模板变量，使用 `{{变量名}}` 格式。
8. 选择或新增标签。
9. 填写修改说明。
10. 点击“保存”。

保存后，系统会自动生成第一条版本记录。

### 4. 搜索和筛选 Prompt

可以通过以下方式查找 Prompt：

- 在搜索框输入关键词，匹配标题、描述、正文、用途、模型建议和标签。
- 点击左侧标签，只查看某一类 Prompt。
- 点击“收藏”，只查看已收藏 Prompt。

建议团队建立统一标签规则，例如：

- 按业务：营销、产品、研发、设计、运营。
- 按场景：调研、写作、评审、分析、复盘。
- 按敏感度：公开、内部、敏感业务。
- 按成熟度：草稿、稳定、最佳实践。

### 5. 使用模板变量

Prompt 正文中可以写：

```text
请为 {{产品名称}} 写一组新品发布文案。目标用户是 {{目标用户}}，核心卖点是 {{核心卖点}}。
```

系统会自动识别变量：

- 产品名称
- 目标用户
- 核心卖点

使用时，在详情页“变量填充”区域填写变量值，然后复制填充后的结果。

适合变量化的内容包括：

- 产品名称
- 行业
- 目标用户
- 输出格式
- 资料内容
- 语气风格
- 字数限制

### 6. 编辑和版本管理

每次编辑 Prompt 后，系统都会生成一个新的历史版本。

建议每次编辑时写清楚修改说明，例如：

- “强化输出格式约束”
- “增加敏感信息脱敏要求”
- “适配 GPT-5.4”
- “压缩为短文案版本”
- “增加表格输出”

在详情页可以：

- 查看所有历史版本。
- 标记某个版本为最佳版本。
- 恢复任意历史版本。
- 对比两个版本的正文差异。

恢复历史版本时，系统会保留新的恢复记录，而不是覆盖掉历史。

### 7. 收藏高质量 Prompt

当某个 Prompt 已经被验证效果稳定，可以点击收藏。

收藏适合用于：

- 团队高频使用模板。
- 经过多次迭代的最佳版本。
- 新成员必须掌握的基础 Prompt。
- 重要业务流程中的标准 Prompt。

### 8. 导出分享包

点击右上角“导入/导出”，选择“生成导出包”。

导出包包含：

- Prompt
- 标签
- 版本历史
- 导出时间
- schemaVersion

导出前建议检查：

- 是否包含客户名称、合同信息、内部策略等敏感内容。
- 是否包含不应跨团队分享的 Prompt。
- 标签命名是否清晰。
- 是否已经标记最佳版本。

### 9. 导入分享包

操作步骤：

1. 点击“导入/导出”。
2. 选择 JSON 分享包。
3. 查看导入预览，包括 Prompt 数量、版本数量、标签数量和冲突数量。
4. 如果存在冲突，选择“跳过重复”或“覆盖重复”。
5. 点击“确认导入”。

导入冲突规则：

- ID 相同会视为冲突。
- 标题相同也会视为冲突。

推荐默认选择“跳过重复”，除非明确知道要用分享包中的版本覆盖本地资产。

### 10. 删除 Prompt

在详情页点击删除按钮，可以删除当前 Prompt。

注意：

- 删除会同时删除该 Prompt 的版本历史。
- 删除前应确认该 Prompt 不再需要复用。
- 如需备份，建议先导出分享包。

### 11. 团队使用建议

为了让 Prompt Vault 真正变成团队资产库，建议建立以下规则：

- 每个 Prompt 必须有清晰标题。
- 每个 Prompt 至少有一个标签。
- 高频 Prompt 应补充用途和模型建议。
- 每次重要修改必须填写修改说明。
- 敏感业务 Prompt 使用单独标签标识。
- 分享前先检查导出包内容。
- 团队定期清理低质量、重复和过期 Prompt。

### 12. 常见问题

#### Q1：这个产品会把 Prompt 上传到云端吗？

不会。当前版本默认使用本地 SQLite 存储，不依赖第三方云服务。

#### Q2：能不能直接调用大模型？

当前版本不做模型调用。产品重点是 Prompt 管理、版本追踪和复用。

#### Q3：多人能不能同时编辑？

当前版本不支持实时多人协作。团队共享通过导入/导出包完成。

#### Q4：Prompt 版本能不能恢复？

可以。详情页的版本历史中可以选择任意版本恢复，恢复动作也会生成新的版本记录。

#### Q5：适合企业敏感 Prompt 吗？

适合本地管理敏感 Prompt，但当前版本还没有加密库、权限审批和审计日志。如果用于更高合规要求的企业场景，建议后续增加本地加密和私有化权限能力。

## 四、推荐落地路径

第一阶段：个人和小团队使用。

- 建立基础 Prompt 资产库。
- 统一标签和命名规则。
- 使用版本记录替代手动复制多个文档。

第二阶段：团队最佳实践沉淀。

- 标记收藏和最佳版本。
- 定期导出分享包。
- 建立 Prompt 评审和清理机制。

第三阶段：私有化团队协作。

- 增加本地库加密。
- 增加私有同步服务。
- 增加权限、审计和团队空间。
- 增加语义搜索和 Prompt 质量评分。
